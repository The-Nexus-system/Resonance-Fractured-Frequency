# Modular Architecture Bible

This document is the architectural source of truth for the Resonance franchise. It defines every reusable room, space, and facility module from which locations in the Resonance universe are assembled. It is not specific to any single location. CSV Hearth is built from these modules, and every future station, colony, ship, settlement, embassy, research outpost, and military installation is built from them as well.

## Purpose

Resonance uses modular architecture. Rooms are reusable. Buildings are reusable. Neighborhoods are reusable. Entire ships are assembled from standardized architectural modules.

The purpose of this document is to let every campaign, chapter, operation, and future production reference modules rather than reinventing rooms. When a location is needed, it is assembled from the modules defined here. When a new kind of room is genuinely required, it is added here once and then reused everywhere, so that the franchise grows a shared architectural language instead of a pile of one-off spaces.

Every module in this document is written so that it answers, for its own space, how it sounds, how it is navigated by players with different bodies and minds, how children and civilians and professionals use it, how it behaves in an emergency, and how it changes when upgraded. Architecture in Resonance is never neutral scenery. Every room is gameplay, story, sound, community, and accessibility at once.

## Canon Rule: A Room Exists Independently of the Ship Containing It

A room exists independently of the ship, station, or settlement that contains it.

Only its location changes. Its purpose remains consistent.

An Engineering Core aboard CSV Hearth and an Engineering Core aboard a later hospital ship are the same module doing the same job in the same recognizable way. A player who has learned to read one Engineering Core by sound and layout can read any Engineering Core, anywhere in the franchise. This consistency is a canon promise to the player and an accessibility guarantee: mastery of a module transfers across every location that uses it.

## How to Read This Document

The document opens with this front matter, then defines each room module as its own section. Every module is written to the same five-part structure so the document stays predictable and screen-reader navigable.

- Overview and Purpose covers what the module is for, how large it typically is, which departments use it, and its primary and secondary activities, along with its narrative purpose.
- Accessibility and Universal Use covers its accessibility features and how blind players, Deaf players, wheelchair users, children, civilians, and professionals each use it, and how it improves accessibility for everyone.
- Soundscape, Lighting, and Environment covers how the module sounds, how it is lit, and the environmental storytelling it makes possible.
- Equipment and Connections covers its typical equipment and how it connects to other modules, including which modules are adjacent, required, and optional.
- Gameplay, Narrative, and Emergencies covers its common NPCs and conversations, the gameplay systems it introduces and reinforces, how it behaves during emergencies, how it changes after being upgraded, and its future reuse opportunities.

Every heading is disambiguated by module name so no two headings collide and any section can be reached directly.

## Modular Design and Connection Points

Modules connect to one another through standardized connection points. Every module presents the same kinds of interfaces: structural mating surfaces, a power tap, an atmosphere and life-support tie-in, a data and communications spine connection, and at least one accessible passage wide enough and clear enough for any body to pass through. Because the connection points are standardized, modules can be recombined freely without redesigning the rooms themselves.

This is what allows entire locations to be assembled from the same architectural language rather than built from scratch. The following are all assembled from these modules:

- CSV Hearth and generation ships like it.
- Orbital stations.
- Planetary settlements.
- Research bases.
- Hospital ships.
- Military vessels.
- Civilian transports.
- Exploration ships.
- Colonies.
- Embassies.
- Locations built by future civilizations that adopt the same standards.

Because every location shares this vocabulary, the player's knowledge of space is itself progression. Learning a module once means understanding it everywhere it appears, in this operation and in every operation to come.

## Bridge

### Bridge — Overview and Purpose

- Purpose: the Bridge is the primary command and coordination module of any vessel or station, the place from which movement, navigation, communications, and whole-ship decisions are directed. It exists independently of the ship containing it; only its location changes, while its purpose of unified command remains consistent.
- Typical size: a large, high-ceilinged central chamber sized to hold a full command watch plus visiting department heads, with a surrounding walkway and standing room for observers during ceremonies or emergencies.
- Departments using it: Command (the Vale System) as the primary occupants; Operations and Logistics for coordination; Engineering and Security for status reporting; Medical during casualty events; Civilian affairs during announcements and festivals.
- Primary activities: navigation and helm control; ship-wide announcements; command decisions; monitoring of all major systems; coordination of departments during normal watches.
- Secondary activities: ceremonies and Founding Day addresses led by Captain Iris and Captain Elian; drills and General Quarters rehearsals led by Captain Naomi; strategic review and analysis sessions led by Captain Gideon; hosting of visiting representatives.
- Narrative purpose: the Bridge is where the player feels the ship as a single living body under a single shared command, and where the four fronting members of the Vale System are recognized by voice rather than by face. It is the emotional center of authority, calm in routine and decisive in crisis.

### Bridge — Accessibility and Universal Use

- Accessibility features: every station is height-adjustable and reachable from a seated position; controls carry tactile markings and consistent physical layout; all visual displays have synchronized audio and haptic equivalents; the floor plan uses continuous guide textures between stations.
- How blind players navigate it: distinct floor textures and directional airflow mark each path; each command station emits a unique quiet signature tone so the space can be mapped by ear; spoken status readouts and a spatial audio overlay describe the position of every active console.
- How Deaf players receive information: all announcements and alerts appear as live captioned text on every surface and on personal displays; Doctor Samira Kade's sleeve-projector conventions are supported ship-wide, showing ASL or live text; visual beacons mirror every audible alarm.
- How wheelchair users move through it: wide, unobstructed approach lanes reach every station; there are no raised command daises requiring steps; Dr. Soren Qadir's repulsor-assisted chair and its manual fallback both clear all thresholds without ramps because thresholds are flush.
- How children use it: a supervised observation ledge with a low rail lets children like Mira Ives watch operations safely; a simplified companion display translates bridge activity into plain language and pictures; AAC input is accepted at a guest console.
- How civilians use it: during addresses and festivals the Bridge opens a public gallery; civilians attend ceremonies, hear announcements at the source, and can submit questions through community terminals.
- How professionals use it: department heads dock their own portable consoles into shared bays, keeping their familiar controls; watch rotation is handled through personal authentication so no one relearns the layout.
- How it naturally improves accessibility for everyone: because every readout is simultaneously spoken, captioned, and haptic, every crew member receives command information in whatever channel suits the moment, and situational awareness rises for all during high-stress operations.

### Bridge — Soundscape, Lighting, and Environment

- Soundscape: a low, steady systems hum underlies calm voices and the soft signature tones of each console; Captain Elian's warm morning announcements, Captain Naomi's crisp emergency cadence, Captain Gideon's measured analytical review, and Captain Iris's welcoming ceremonial address are all recognizable by voice from the same body's footsteps.
- Lighting: broad, even, glare-free illumination in normal watches; a gentle amber shift for evening watch; a distinct, non-strobing pulse pattern reserved for General Quarters that is safe for photosensitive crew such as Chief Petty Officer Jonah Briggs.
- Environmental storytelling opportunities: worn armrests on the command chair hint at long watches; a Founding Day banner half-raised for the festival that never happens; personal tokens left at department consoles; the shared command chair quietly signaling that four people steward one body.

### Bridge — Equipment and Connections

- Typical equipment: helm and navigation consoles; ship-wide communications array; master systems status board with audio and haptic mirrors; docking bays for portable department consoles; the shared command chair; captioning and beacon relays.
- Adjacent modules: Ready Room; Operations Center; Lift Lobby; Primary Corridor.
- Required modules: Communications Center; Power Distribution; Life Support.
- Optional modules: Observation Dome; Emergency Coordination Center; Ready Room suite expansions for larger vessels.

### Bridge — Gameplay, Narrative, and Emergencies

- Common NPCs: the fronting member of the Vale System on watch; Chief Petty Officer Jonah Briggs relaying operations; rotating department representatives; occasionally Mira Ives on the observation ledge.
- Typical conversations: watch handovers; navigation and course debate; announcement drafting with Captain Iris or Captain Elian; strategic review with Captain Gideon; drill briefings with Captain Naomi.
- Gameplay systems introduced: command network access; ship-wide coordination; navigation planning; the recognition-by-voice system for the Vale System members.
- Gameplay systems reinforced: department status monitoring; emergency command escalation; large-scale coordination across rescued crew.
- Emergency behavior: during General Quarters the Bridge shifts to Captain Naomi's command cadence, non-essential displays dim, alert beacons and captions synchronize, and department consoles lock to crisis priorities.
- How it changes after being upgraded: upgrades add finer sensor integration, redundant communications, expanded captioning throughput, and additional docking bays so more rescued specialists can coordinate at once.
- Future reuse opportunities: the same Bridge module anchors orbital stations, exploration ships, and embassies, always serving as the single point of unified command wherever it is installed.

### Ready Room — Equipment and Connections

## Engineering Core

### Engineering Core — Overview and Purpose

- Purpose: the Engineering Core is the central power and propulsion module that generates, routes, and balances the energy every other system depends on. It exists independently of any ship; only its location changes while its purpose of powering the whole remains consistent.
- Typical size: a tall, multi-level chamber built around the main reactor and generator assembly, with catwalks, diagnostic alcoves, and a central power-routing gallery.
- Departments using it: Engineering as primary, led by Chief Engineering Systems Specialist Mara Chen; Maintenance for repairs; Command for status; Security during containment events.
- Primary activities: power generation and routing; generator management; advanced diagnostics; energy balancing; sensor calibration.
- Secondary activities: upgrade installation; engineering training; acoustic diagnostic teaching; coordination with Maintenance on preventative work.
- Narrative purpose: the Engineering Core is where the player learns that engineering can be understood through sound, where Mara diagnoses the ship's health by ear, and where the heartbeat of the vessel is most audible.

### Engineering Core — Accessibility and Universal Use

- Accessibility features: the entire core is designed to be diagnosed and operated by sound as a first-class channel; controls are tactile and consistent; all levels are reachable without stairs via flush ramps and lifts; every alarm is audible, visible, and haptic.
- How blind players navigate it: the core is the flagship example of acoustic navigation, with each system emitting a distinct, meaningful tone; guide textures and airflow mark every catwalk; Mara's diagnostic method is the intended play pattern, mapping the space and its health entirely by ear.
- How Deaf players receive information: every system tone has a synchronized visual and haptic waveform display, so the acoustic diagnostics Mara performs by ear are fully available as sight and touch; captioned alerts mirror all audible cues.
- How wheelchair users move through it: catwalks and galleries are wide and flush, all levels are served by lifts, and diagnostic stations have seated clearance; Dr. Soren Qadir's chair, including its manual fallback, reaches every working position.
- How children use it: a protected education alcove lets children safely watch and listen to the reactor's rhythm and learn the meaning of its sounds; AAC prompts explain what each tone means.
- How civilians use it: normally restricted, but civilians attend supervised education tours and can access an explanatory public feed of the ship's power health.
- How professionals use it: engineers dock portable diagnostic tools that adopt the core's tonal language; Maintenance staff coordinate repairs from shared alcoves.
- How it naturally improves accessibility for everyone: because the core presents its state simultaneously as sound, sight, and touch, every crew member can sense the ship's health, and the acoustic-first design pioneered for Mara becomes a shared literacy across departments.

### Engineering Core — Soundscape, Lighting, and Environment

- Soundscape: a rich, layered signature of generator cycling, power relays, and a stable electrical hum, each pitch carrying diagnostic meaning; the whole core is tuned so that a healthy ship sounds harmonious and a faulty one sounds discordant.
- Lighting: cool, even illumination with clear waveform displays; a steady, non-strobing alert pattern reserved for containment events that is safe for photosensitive crew.
- Environmental storytelling opportunities: a listening chair worn smooth where Mara sits to hear the ship; tactile relay maps rubbed shiny by fingers; a subtle change in the core's tone that foreshadows trouble before any gauge moves.

### Engineering Core — Equipment and Connections

- Purpose: The Library is the quiet, browsable home of the ship's accumulated texts, records, and personal reading, a place to find, read, and think without being rushed.
- Typical size: Medium, with a mix of open reading floor, private nooks, and dense shelving or data stacks along the perimeter.
- Departments using it: Knowledge Archive staff maintain it, but every department and every civilian uses it for reference, study, and leisure reading.
- Primary activities: Reading, research, quiet study, and retrieval of physical and archived materials.
- Secondary activities: Small tutoring sessions, reading circles for children, and personal writing or correspondence.
- Narrative purpose: The Library is the ship's long memory in a human, unhurried form, where the player can piece together history, character backstory, and canon at their own pace.

### Library — Accessibility and Universal Use

- Accessibility features: Aisles are wide and consistent, shelving tops out at reachable height with lower reserves for anything taller, and every physical item has an audio and Braille label counterpart.
- How blind players navigate it: A consistent grid of tactile floor guides and audio shelf beacons lets a blind reader find a section by sound and touch, and any text can be read aloud on request through a personal listening point.
- How Deaf players receive information: All spoken assistance is mirrored as text on reading terminals, and staff communicate through visible text and signing rather than raised voices, keeping the room genuinely quiet.
- How wheelchair users move through it: The full grid of aisles is level and turn-friendly, reading tables include knee clearance, and retrieval assistance brings high items down so no shelf is out of reach.
- How children use it: A dedicated low-shelf children's corner with soft seating sits within sightline of the main desk, and tactile picture materials serve early readers and non-verbal children alike.
- How civilians use it: Civilians borrow freely, reserve nooks, and contribute personal writings to the community shelves, treating the Library as shared living-room space rather than an institution.
- How professionals use it: Professionals pull technical references, cross-check Archive entries, and reserve quiet nooks for focused problem-solving away from their workstations.
- How it naturally improves accessibility for everyone: Because every item is available as text, audio, and Braille by default, the same catalog serves a tired reader, a commuting listener, and a blind researcher identically.

### Library — Soundscape, Lighting, and Environment

- Soundscape: A hushed, softly padded quiet with the faint rustle of pages, the low hum of data stacks, and gentle acoustic damping that swallows sudden noise so the room stays calm.
- Lighting: Warm, glare-free reading light at each table and nook, dimmer in the stacks, brighter over reading surfaces, all adjustable at the individual seat.
- Environmental storytelling opportunities: Hand-annotated margins, a shelf of well-worn favorites, and a memorial reading nook dedicated to a lost crew member let the player read the ship's emotional history between the lines.

### Library — Equipment and Connections

- Typical equipment: Physical and archived shelving, personal listening and reading terminals, Braille embossers, adjustable reading lamps, and a staffed retrieval desk.
- Adjacent modules: Sits beside the Knowledge Archive and the Classroom so reference, deep archive, and instruction connect naturally.
- Required modules: Requires Power Distribution for terminals and Communications Center to sync with the Knowledge Archive catalog.
- Optional modules: Optionally links to the Cafe for a reading-and-refreshment corner and to the Meditation Room for adjoining quiet.

### Library — Gameplay, Narrative, and Emergencies

- Common NPCs: Archive librarians, quiet studying crew, and Mira Ives when she visits for tactile picture materials and her preferred calm corner.
- Typical conversations: Requesting a hard-to-find record, discussing a shared reading, asking for an audio version, or trading personal writings with a neighbor.
- Gameplay systems introduced: Introduces the personal research and lore-collection loop, where discovered documents deepen the player's understanding of canon.
- Gameplay systems reinforced: Reinforces the Knowledge Archive system and character-relationship building through shared reading and quiet company.
- Emergency behavior: During emergencies the Library becomes a designated calm shelter for civilians and children, lighting steadies, and staff guide occupants along the tactile grid to the nearest safe route.
- How it changes after being upgraded: Upgrades expand the archived catalog, add faster cross-ship retrieval, and grow the accessible-format library so more items exist in every format.
- Future reuse opportunities: The module reappears as colony public libraries, research-base reference rooms, and shipboard reading spaces, always a place for unhurried thought.

## Community Hall

### Community Hall — Overview and Purpose

- Purpose: The Community Hall is the ship's largest flexible gathering space, an open floor that reconfigures for meetings, festivals, markets, votes, and shared ceremonies.
- Typical size: Large, an open high-ceilinged room with stackable seating and movable partitions to scale from an intimate circle to a full-ship assembly.
- Departments using it: Command and Civilian affairs host here most often, with every department contributing to festivals, drills briefings, and community votes.
- Primary activities: Town gatherings, festivals, community votes, markets, and shared meals for special occasions.
- Secondary activities: Rehearsals, exhibitions, blood-and-supply drives, and overflow space for events that outgrow their home rooms.
- Narrative purpose: The Community Hall is where the ship acts as one people. It is where Founding Day was meant to be celebrated, making its emptiness on Attack Day one of the campaign's heaviest silences.

### Community Hall — Accessibility and Universal Use

- Accessibility features: A single level floor with no thresholds, distributed hearing loops, movable seating that always preserves clear paths, and portable signing and captioning stations.
- How blind players navigate it: Permanent tactile edge paths and audio zone beacons stay fixed even when the furniture moves, so the room's shape is always readable by sound and touch.
- How Deaf players receive information: Large mobile caption screens and a lit signing platform accompany every address, and event schedules are posted as clear visual and tactile signage.
- How wheelchair users move through it: The flat open floor is fully rollable, seating is arranged with wide lanes, and raised platforms for speakers all include ramps.
- How children use it: Children have designated floor-seating and play-adjacent zones at events, and festival activities are staged at low heights so they participate rather than merely watch.
- How civilians use it: This is the civilians' primary shared room, bookable for any community purpose, and it holds the ship's votes so every resident has a physical place in decisions.
- How professionals use it: Professionals brief the whole community here, run coordinated drills, and stage cross-department exhibitions that would not fit their own modules.
- How it naturally improves accessibility for everyone: Because the room is designed to be reconfigured while keeping fixed accessible paths and services, no rearrangement ever accidentally excludes anyone.

### Community Hall — Soundscape, Lighting, and Environment

- Soundscape: A large, gently reverberant room whose absorptive ceiling keeps a full crowd intelligible; during quiet hours it carries a soft distant hum that makes its emptiness feel large.
- Lighting: Broad adjustable house lighting that shifts from bright market daylight to warm festival glow, with clearly lit speaking and signing zones.
- Environmental storytelling opportunities: Festival banners half-hung, a Founding Day countdown left frozen, and community mural walls updated each year let the player feel both celebration and interrupted celebration.

### Community Hall — Equipment and Connections

- Typical equipment: Stackable seating, movable partitions, portable stages, distributed loops, mobile caption and signing stations, and a public address and voting system.
- Adjacent modules: Sits at a Transit Hub and beside the Dining Hall so crowds arrive and disperse easily and shared meals flow between the two.
- Required modules: Requires Power Distribution and Communications Center for its address, captioning, and voting systems.
- Optional modules: Optionally links to the Performance Hall, the Kitchen for catered events, and the Memorial Wall for ceremonies of mourning.

### Community Hall — Gameplay, Narrative, and Emergencies

- Common NPCs: Captain Iris of the Vale System hosting civilian gatherings and festivals, community organizers, market vendors, and residents of every department.
- Typical conversations: Casting a community vote, organizing a festival, hearing a captain's address, or trading news and goods at market.
- Gameplay systems introduced: Introduces the community-decision and festival systems, where the player influences ship-wide morale and choices.
- Gameplay systems reinforced: Reinforces relationship and reputation systems as the player is recognized across the whole community at once.
- Emergency behavior: During General Quarters the Hall becomes a primary muster and headcount point; partitions retract, the address system switches to Captain Naomi of the Vale System, and clear evacuation lanes open along the fixed tactile paths.
- How it changes after being upgraded: Upgrades add faster reconfiguration, richer festival staging, and ship-wide broadcast so a gathering can include those who cannot attend in person.
- Future reuse opportunities: The module returns as colony town halls, station assembly rooms, and embassy reception spaces, always the room where a community gathers as one.

## Dining Hall

### Dining Hall — Overview and Purpose

- Purpose: The Dining Hall is the shared eating space where the ship gathers by shift to be fed together, a daily anchor of routine and community.
- Typical size: Large, with long communal tables, smaller quiet tables, and an open serving line fed directly from the adjacent Kitchen.
- Departments using it: Logistics and Kitchen staff run it, and every department and civilian passes through across the day's meal shifts.
- Primary activities: Serving and eating meals, informal meetings over food, and the ordinary social rhythm of shared tables.
- Secondary activities: Study and work over long meals, small celebrations, and staging point for festival feasts.
- Narrative purpose: The Dining Hall is where the ship's ordinary life is most visible; who sits with whom, who eats alone, and which chairs are empty tells the player the state of the community.

### Dining Hall — Accessibility and Universal Use

- Accessibility features: A mix of fixed and movable seating at varied heights, level serving-line access, trays and utensils designed for one-handed and low-grip use, and clear allergen and ingredient labeling in multiple formats.
- How blind players navigate it: A tactile path leads from entrance to serving line to seating, the serving line is announced by station, and menu items are read aloud and offered in Braille.
- How Deaf players receive information: Menus, shift times, and announcements appear on visible screens, and serving staff confirm orders through text and gesture rather than calling out.
- How wheelchair users move through it: The serving line has a lowered reachable section, tables include knee clearance, and lanes between tables stay wide enough to turn and pass.
- How children use it: Lower tables, child-height serving access, and spill-tolerant surfaces let children serve and seat themselves, and family tables keep parents and children together.
- How civilians use it: Civilians eat here as their default, host small celebrations, and mix freely with crew, making the Hall the most integrated room on the ship.
- How professionals use it: Professionals grab quick shift meals, hold informal working lunches, and use the Hall as neutral ground for cross-department conversation.
- How it naturally improves accessibility for everyone: Because varied seating, reachable serving, and clear labeling are standard, everyone from a child to a blind diner to a crew member on a short break is served the same easy way.

### Dining Hall — Soundscape, Lighting, and Environment

- Soundscape: A warm busy hum of conversation, clinking trays, and the serving line, tuned by absorptive surfaces so it stays lively without becoming overwhelming; quiet corners are acoustically softer.
- Lighting: Bright and welcoming over the serving line, warmer and softer over the tables, with calmer low-stimulation lighting in a designated quiet section.
- Environmental storytelling opportunities: A rotating festival menu board, reserved memorial seats, and children's artwork on the walls let the player read the community's mood in its most everyday room.

### Dining Hall — Equipment and Connections

- Typical equipment: A serving line with warming and chilling stations, communal and quiet tables, tray-return systems, allergen-labeling displays, and beverage points.
- Adjacent modules: Directly connected to the Kitchen and near the Cafe and Community Hall to form the ship's central social cluster.
- Required modules: Requires the Kitchen for food, Water Processing, and Power Distribution.
- Optional modules: Optionally links to Hydroponics for fresh produce and to the Community Hall for festival feasts.

### Dining Hall — Gameplay, Narrative, and Emergencies

- Common NPCs: Chief Petty Officer Jonah Briggs coordinating shift logistics, kitchen and serving staff, and a shifting cross-section of the whole crew and civilian population.
- Typical conversations: Catching up with a crew member, overhearing ship news, organizing a small gathering, or checking on someone who has been eating alone.
- Gameplay systems introduced: Introduces the social-hub and relationship-check-in loop, where shared meals build trust and reveal character state.
- Gameplay systems reinforced: Reinforces logistics and morale systems as meal supply and community mood connect to the wider ship economy.
- Emergency behavior: During emergencies the Dining Hall becomes a rapid feeding and coordination point for responders; the serving line switches to ration handout, and its wide lanes route people toward muster stations.
- How it changes after being upgraded: Upgrades expand menu variety, add faster serving throughput, and improve the quiet section so more diners can be comfortable at once.
- Future reuse opportunities: The module returns as colony mess halls, station canteens, and hospital-ship dining rooms, always the daily table a community shares.

## Kitchen

### Kitchen — Overview and Purpose

- Purpose: The Kitchen is the production space where the ship's meals are prepared and portioned, feeding the adjacent Dining Hall and catering community events.
- Typical size: Medium, a dense working room of prep stations, cooking lines, cold and dry storage, and a wash area.
- Departments using it: Logistics and food-service staff run it, with Medical consulting on nutrition and allergens and Hydroponics supplying fresh produce.
- Primary activities: Meal preparation, cooking, portioning, and cleaning across meal shifts.
- Secondary activities: Menu planning, preserving and storing surplus, and preparing festival and ceremonial dishes.
- Narrative purpose: The Kitchen is the ship's quiet engine of care; feeding a whole civilization every day is a constant, unglamorous act of keeping people alive and together.

### Kitchen — Accessibility and Universal Use

- Accessibility features: Height-adjustable prep counters, front-mounted and lever controls on all equipment, clearly marked hot and sharp zones, and a consistent tool-placement discipline so nothing moves unexpectedly.
- How blind players navigate it: Fixed station layout, tactile edge and hazard markers, spoken equipment status, and an unbreakable rule that tools return to marked homes let a blind cook work safely by memory and touch.
- How Deaf players receive information: Timers and equipment alerts are visual and vibrating, hazard states show as lights, and staff coordinate through gesture and visible order screens.
- How wheelchair users move through it: Roll-under prep counters, reachable cooktop controls, and pull-down storage let a wheelchair user run a full station, with clear turning space in the working lanes.
- How children use it: Supervised children help at a dedicated low, cool prep station with blunt tools, learning food skills without entering hot or sharp zones.
- How civilians use it: Civilian volunteers assist during festivals and busy shifts at clearly bounded safe stations, keeping food a shared community effort.
- How professionals use it: Food-service professionals run the cooking lines and coordinate supply with Logistics, using standardized stations that any trained crew member can step into.
- How it naturally improves accessibility for everyone: Because hazards are marked, tools have fixed homes, and controls are reachable and front-mounted, the room is safer and clearer for a tired cook, a new volunteer, and a blind chef alike.

### Kitchen — Soundscape, Lighting, and Environment

- Soundscape: A rhythmic working clatter of chopping, sizzling, running water, and distinct equipment tones, each device given a recognizable sound so it can be tracked without sight.
- Lighting: Bright, shadow-free task lighting over every station with clearly lit hazard zones, tuned to reveal detail without glare on wet surfaces.
- Environmental storytelling opportunities: A worn favorite knife block, a wall of handwritten recipes from many cultures, and a paused festival prep board show generations of shared cooking and an interrupted celebration.

### Kitchen — Equipment and Connections

- Typical equipment: Adjustable prep counters, cooking lines, cold and dry storage, wash stations, portioning tools, and vibrating-visual timers and alerts.
- Adjacent modules: Directly connected to the Dining Hall and near Hydroponics and Cargo Bay for supply.
- Required modules: Requires Water Processing, Power Distribution, and Life Support ventilation.
- Optional modules: Optionally links to Hydroponics for fresh produce, the Cafe, and the Community Hall for catered events.

### Kitchen — Gameplay, Narrative, and Emergencies

- Common NPCs: Food-service staff, Chief Petty Officer Jonah Briggs coordinating supply and rationing, and civilian volunteers during busy periods.
- Typical conversations: Planning a menu, sourcing an ingredient, adapting a dish for an allergy, or organizing a festival feast.
- Gameplay systems introduced: Introduces the food-production and supply-management loop, where ingredients, labor, and demand must balance.
- Gameplay systems reinforced: Reinforces logistics, resource management, and Hydroponics production chains.
- Emergency behavior: During emergencies the Kitchen shifts to rapid ration production, hot equipment is safely locked down, and output is routed to the Dining Hall handout line and responder stations.
- How it changes after being upgraded: Upgrades add faster equipment, expanded storage, and preservation systems that stretch supply through shortages.
- Future reuse opportunities: The module returns as colony kitchens, station galleys, and hospital-ship food-production rooms, always the place that feeds the community.

## Hydroponics

### Hydroponics — Overview and Purpose

- Purpose: Hydroponics is the ship's food-and-air growing facility, cultivating crops in soilless beds to feed the crew and help regenerate the atmosphere.
- Typical size: Large, tiered growing racks and troughs arranged in long aisles under tuned grow lighting.
- Departments using it: Life-support and agriculture staff run it, with the Kitchen drawing produce and Research studying crop performance.
- Primary activities: Growing, tending, harvesting, and cycling crops, and contributing oxygen and humidity to Life Support.
- Secondary activities: Seed banking, crop research, teaching plant care, and providing calming green space.
- Narrative purpose: Hydroponics is proof that the ship is alive and self-sustaining; its rows of growing food are hope made physical, and any threat to it raises the campaign's stakes.

### Hydroponics — Accessibility and Universal Use

- Accessibility features: Growing beds at multiple heights including roll-under troughs, tactile row labels, level non-slip aisles, and tool stations with adaptive grips.
- How blind players navigate it: Consistent aisle numbering, tactile and scented row markers, audio system-status cues, and the plants' own smells and sounds make the space richly navigable without sight.
- How Deaf players receive information: Environmental readouts, irrigation and climate alerts, and harvest schedules are all visual, and status lights indicate each zone's condition.
- How wheelchair users move through it: Roll-under growing troughs, reachable controls, and wide level aisles let a wheelchair user tend crops fully hands-on.
- How children use it: A children's growing patch with low beds and simple tasks teaches plant care and lets children literally help feed the ship.
- How civilians use it: Civilians volunteer on growing shifts and keep small personal plots, sharing responsibility for the ship's food.
- How professionals use it: Agriculture and life-support professionals manage the growing cycles and climate systems, and Research runs crop trials in dedicated beds.
- How it naturally improves accessibility for everyone: Because beds sit at varied heights and rows are marked by touch, scent, and sound, tending plants is possible for a child, a blind volunteer, and a seated grower alike.

### Hydroponics — Soundscape, Lighting, and Environment

- Soundscape: A living hum of circulation pumps, trickling irrigation, the whir of climate fans, and the soft rustle of leaves, with each system's tone distinct enough to diagnose by ear.
- Lighting: Bright tuned grow-spectrum light over the beds, shifting warmer or cooler by crop, with softer working light in the aisles.
- Environmental storytelling opportunities: A heritage-seed shelf, a struggling experimental crop row, and a memorial planting for a lost crew member give the growing space memory and stakes.

### Hydroponics — Equipment and Connections

- Typical equipment: Tiered growing beds and troughs, irrigation and nutrient systems, grow lighting, climate control, seed banks, and harvest tools with adaptive grips.
- Adjacent modules: Sits near Life Support and the Kitchen and connects to Water Processing and Recycling for its inputs.
- Required modules: Requires Water Processing, Recycling Center, Power Distribution, and Life Support integration.
- Optional modules: Optionally links to the Botanical Garden, the Research Laboratory, and the Materials Lab.

### Hydroponics — Gameplay, Narrative, and Emergencies

- Common NPCs: Agriculture and life-support staff, civilian growing volunteers, and Dr. Soren Qadir when studying crop biology from his powered chair.
- Typical conversations: Planning a harvest, troubleshooting a failing crop, trading produce with the Kitchen, or teaching a child to plant.
- Gameplay systems introduced: Introduces the crop-cultivation and food-and-air production loop tied to ship sustainability.
- Gameplay systems reinforced: Reinforces Life Support, resource management, and Kitchen supply chains.
- Emergency behavior: During emergencies Hydroponics prioritizes protecting the growing beds and seed bank, seals compromised zones to preserve atmosphere, and its climate systems can be redirected to support Life Support.
- How it changes after being upgraded: Upgrades add higher-yield beds, resilient crop strains, and finer climate control that stabilizes the whole ship's atmosphere.
- Future reuse opportunities: The module returns as colony farms, station agricultural bays, and exploration-ship greenhouses, always the place that grows food and air.

## Child Care

### Child Care — Overview and Purpose

- Purpose: Child Care is the supervised space where young children are looked after, taught, and nurtured while their parents work, integrated into the ship's daily life.
- Typical size: Medium, divided into activity, rest, and quiet zones with a secure entrance.
- Departments using it: Child-care and education staff run it, with Medical and wellness support, serving families across every department.
- Primary activities: Supervised care, early learning, structured and free play, meals, and rest for young children.
- Secondary activities: Parent consultations, developmental observation, and coordination with the Classroom and Playground.
- Narrative purpose: Child Care shows that the ship protects its most vulnerable as a community responsibility; it embodies the colony's investment in its future.

### Child Care — Accessibility and Universal Use

- Accessibility features: Everything at child height, varied-ability play and learning materials, a calm sensory-regulation zone, secure controlled access, and staff trained in every child's communication method.
- How blind players navigate it: Consistent zone layout, tactile and scented area markers, and sound cues let a blind child move confidently, and staff describe activities aloud.
- How Deaf players receive information: Staff sign fluently, visual schedules and cues structure the day, and a Deaf child communicates and learns with no barrier.
- How wheelchair users move through it: Level open zones, roll-under activity tables, and reachable materials let wheelchair-using children take part in everything.
- How children use it: The whole module is built for children, with zones scaled to their bodies and materials chosen so every child, including autistic, ADHD, and non-verbal children, is supported.
- How civilians use it: Civilian parents drop off and collect children, consult with staff, and volunteer, sharing the community's care.
- How professionals use it: Child-care and education professionals deliver care and early learning, and Medical and wellness staff advise on individual needs, including Mira's AAC system.
- How it naturally improves accessibility for everyone: Because staff are fluent in many communication methods and materials suit every ability, every child is met as they are, teaching the whole community that difference is ordinary.

### Child Care — Soundscape, Lighting, and Environment

- Soundscape: A gentle blend of quiet play, soft songs, and calm voices, deliberately kept from overload, with a genuinely hushed rest and regulation zone.
- Lighting: Soft, warm, adjustable light, dimmable in the rest zone and calm in the sensory area, never harsh or flickering.
- Environmental storytelling opportunities: Children's artwork, a wall of handprints across the years, and a comfort corner with well-loved toys give the room deep continuity and warmth.

### Child Care — Equipment and Connections

- Typical equipment: Child-height furniture, adaptive learning and play materials, rest mats, a sensory-regulation station, secure access controls, and AAC-compatible devices.
- Adjacent modules: Sits beside the Playground and the Classroom and near Residential Corridors within the family cluster.
- Required modules: Requires Power Distribution, Life Support, and secure adjacency to the Playground and Classroom.
- Optional modules: Optionally links to the Medical Reception, the Botanical Garden, and the Community Workshop.

### Child Care — Gameplay, Narrative, and Emergencies

- Common NPCs: Child-care staff, young children, visiting parents, and Rowan Ives dropping off or collecting Mira and comparing notes with staff on her needs.
- Typical conversations: A parent's reassurance, a staff developmental update, arranging care schedules, or discussing a child's communication needs.
- Gameplay systems introduced: Introduces the child-care and early-development support loop that ties into family and community wellbeing.
- Gameplay systems reinforced: Reinforces community responsibility, accessibility, and relationship systems.
- Emergency behavior: During General Quarters staff immediately secure and account for every child, the module becomes a protected shelter, and children are reunited with families along safe routes without separation.
- How it changes after being upgraded: Upgrades add richer learning materials, an expanded sensory-regulation zone, and improved individual-needs support.
- Future reuse opportunities: The module returns as colony nurseries, station family-care centers, and settlement childcare spaces, always where the youngest are protected.

## Community Workshop

- Typical size: Medium, an open floor of shared workbenches, tool walls, and project storage.
- Departments using it: Open to all residents; loosely stewarded by Maintenance and education staff who keep tools and teach skills.

- Narrative purpose: The Community Workshop embodies the ship's ethos that everyone can contribute and make; it is where civilians become capable and where cross-department friendships form over shared projects.

### Community Workshop — Accessibility and Universal Use

- How wheelchair users move through it: Roll-under benches, reachable tool storage, and wide level lanes let a wheelchair user run a full project independently.
- How children use it: A supervised low workbench with safe tools lets children make and repair alongside adults, learning craft as a normal part of ship life.

- How professionals use it: Off-duty professionals mentor here and lend expertise, and Maintenance staff run skill-sharing sessions so survival craft spreads through the community.
- How it naturally improves accessibility for everyone: Because adaptive tools, reachable benches, and marked hazards are standard, a child, a blind maker, and a seated crafter all work the same shared space safely.

### Community Workshop — Soundscape, Lighting, and Environment

- Soundscape: A varied working mix of hand tools, light machinery, and conversation, each tool voiced distinctly and kept from becoming overwhelming by absorptive surfaces and bounded noisy zones.
- Lighting: Bright, shadow-free task lighting at every bench with clearly lit hazard areas and adjustable brightness per station.
- Environmental storytelling opportunities: A wall of finished community projects, a lost-and-found of half-built pieces, and donated heirloom tools show a culture of shared making across generations.

### Community Workshop — Equipment and Connections

- Typical equipment: Shared adjustable workbenches, hand and light power tools with adaptive grips, project storage, material bins, and visual-vibrating safety alerts.
- Adjacent modules: Sits near the Maintenance Workshop, the Fabrication Shop, and Residential Corridors so materials and expertise flow between professional and community making.
- Required modules: Requires Power Distribution, Life Support ventilation, and material supply from Cargo Bay.
- Optional modules: Optionally links to the Maintenance Workshop, the Fabrication Shop, the Playground, and the Community Hall.

- Typical conversations: Asking for help with a build, teaching a technique, planning a community project, or borrowing a tool.
- Gameplay systems introduced: Introduces the community-crafting and skill-sharing loop where residents gain practical capabilities.
- Gameplay systems reinforced: Reinforces fabrication, repair, resource-recovery, and relationship systems.

- Future reuse opportunities: The module returns as colony maker spaces, station community shops, and settlement repair hubs, always where residents build together.

## Music Room

- Purpose: a dedicated space for making, practicing, and sharing music, built so that anyone can play regardless of body, hearing, or experience.

### Music Room — Accessibility and Universal Use

- How professionals use it: therapists run rhythm and breathing sessions, educators teach structured lessons, and audio organizers stage rehearsals for larger performances.
- How it naturally improves accessibility for everyone: because rhythm is delivered visually, tactilely, and audibly at once, every player experiences music through more than one sense, which makes the room richer for all rather than adapted for some.

### Music Room — Soundscape, Lighting, and Environment

- Soundscape: warm layered practice tones, the soft thrum of tuned strings and reeds, overlapping fragments of melody, and a comfortable acoustic that never turns harsh.
- Lighting: soft, adjustable, and warm, dimmable for focused practice and raised gently for lessons, with visual tempo bars glowing along one wall.
- Environmental storytelling opportunities: instruments left mid-tuning, a child's practice chart pinned to the wall, an empty chair kept ready for a musician not yet aboard, and repaired instruments that show the community's care.

### Music Room — Equipment and Connections

- Typical equipment: adjustable instruments, tactile floor plates, tuning tools, visual tempo displays, sheet-music panels in text, braille, and audio, and a small repair bench.
- Adjacent modules: Recording Studio, Performance Hall, and Community Hall.
- Required modules: Power Distribution and Life Support.
- Optional modules: Recording Studio for capturing sessions and Child Care for supervised young learners.

### Music Room — Gameplay, Narrative, and Emergencies

- Common NPCs: community musicians, an audio organizer, visiting children, and occasionally Mira Ives exploring sound through her AAC-linked rhythms.
- Typical conversations: lesson exchanges, invitations to rehearse, gentle encouragement to try an instrument, and talk of an upcoming performance.
- Gameplay systems introduced: rhythm and tempo interactions, and the sound-as-progression cue where new instruments join the mix as crew arrive.
- Gameplay systems reinforced: multi-sensory listening skills and the player's growing recognition of characters by sound.
- Emergency behavior: instruments are secured, tactile floor plates switch to conveying alert patterns, and the room becomes a quiet muster point where vibration and light relay instructions.
- How it changes after being upgraded: gains more instrument stations, finer tactile resolution, and a link to the Recording Studio for shared performances.
- Future reuse opportunities: appears in any station, colony, or ship that maintains a cultural life, and scales from a single practice alcove to a full conservatory wing.

- Accessibility features: raised tactile lettering, braille name panels, audio tribute stations, captioned recordings, and reachable panels at seated and standing heights.
- How blind players navigate it: every name is rendered in raised tactile lettering and braille, and audio stations speak each person's story so the Wall is fully readable by touch and sound.
- How Deaf players receive information: all audio tributes are captioned and presented as text, and visual tribute displays accompany each name.
- How wheelchair users move through it: names and tribute controls are placed within seated reach along the full length, with a level standing-and-viewing area.
- How children use it: lower panels and simple spoken tributes let children learn the names and stories at their own level; Mira Ives can engage through tactile lettering at her own pace.
- How civilians use it: an open place to find a loved one's name, touch it, and hear their story preserved.
- How professionals use it: Archive staff maintain and cross-reference records through accessible, multi-sensory entry stations.
- How it naturally improves accessibility for everyone: because every name exists in touch, sound, and sight at once, no person is ever cut off from remembering someone they lost.

### Memorial Wall — Soundscape, Lighting, and Environment

- Soundscape: a reverent hush, the soft murmur of recorded tributes at listening stations, and the quiet footsteps of those paying respects.
- Lighting: soft and dignified, with gentle illumination guiding hands to names and warm accent light during dedications.
- Environmental storytelling opportunities: worn-smooth names touched by countless hands, fresh tributes beside decades-old ones, and a growing section that fills after the attack.

### Memorial Wall — Equipment and Connections

- Typical equipment: tactile and braille name panels, audio tribute stations, captioned tribute displays, and archival record links.
- Adjacent modules: Memorial Garden, Archive, Community Hall, and a Primary Corridor position.
- Required modules: Archive link for records and Power Distribution.
- Optional modules: Multi-Faith Reflection Room, Meditation Room, and Observation Dome.

### Memorial Wall — Gameplay, Narrative, and Emergencies

- Common NPCs: civilians paying respects, an archivist maintaining records, and Captain Iris of the Vale System leading a dedication.
- Typical conversations: recovering a lost person's story, adding a tribute, and reflecting on those who came before.
- Gameplay systems introduced: memorial record-keeping and story-recovery.
- Gameplay systems reinforced: knowledge archiving, community relationships, and emotional narrative pacing.
- Emergency behavior: during General Quarters the corridor housing the Wall remains a clear, guided evacuation route, and the Wall itself is protected in place as a preserved record.
- How it changes after being upgraded: upgrades add capacity for new names, richer recorded tributes, and deeper archival cross-references.
- Future reuse opportunities: every community across the franchise reuses this module to name and preserve its dead wherever memory must endure.

## End State

This document is the definitive architectural reference for the Resonance franchise. Future locations are created by assembling these existing modules rather than inventing new rooms. When a genuinely new module is required, it is added here once, written to the same five-part structure, and then reused everywhere, so the franchise's architectural language grows coherently instead of fragmenting into one-off spaces.

Because every module specifies how it sounds, how it is navigated by every kind of body and mind, how it serves children, civilians, and professionals, how it behaves in an emergency, and how it changes when upgraded, any campaign that assembles a location from these modules inherits its gameplay, story, sound, community, and accessibility already intact.
