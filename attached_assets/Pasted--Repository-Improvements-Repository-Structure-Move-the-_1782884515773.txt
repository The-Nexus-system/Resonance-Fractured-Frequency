## Accessible Input

Every important action must support multiple activation methods.

Examples include:

- Double tap

- Shake device

- iOS Back Tap

- Android Quick Tap (when available)

- Custom gesture

- External switch device

- Game controller

- Keyboard

- Braille display commands

- Voice commands (where appropriate)

Players may bind any gameplay action to any supported input.

Examples include:

- Fire

- Interact

- Retrieve object

- Scan

- Reload

- Repeat Faience

- Pause

- Accessibility menu

No action should ever depend upon one mandatory gesture.

---

## Tactile Language

Do not simply convert music into vibration.

Instead, create an intentional tactile language.

Use authored vibration patterns to communicate:

- Direction

- Threat

- Distance

- Object proximity

- AI urgency

- Environmental hazards

- Navigation

- Success

- Failure

Music may influence tactile patterns, but haptics should be designed as an independent communication channel.

From this point forward:
